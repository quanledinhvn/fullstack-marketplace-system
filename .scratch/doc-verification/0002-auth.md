## What to build

End-to-end authentication: backend login endpoint, header-based auth guard, database seed, and React login page with role-based redirect.

**API (header-based auth, no JWT):**

- `POST /auth/login` — accepts `{ email, password }`, returns `{ userId, role, name, email }`
- `UserIdGuard` reads `Authorization: <userId>` header on all subsequent requests, queries DB, attaches `@CurrentUser()`
- `@Public()` decorator exempts `/auth/login`
- `RolesGuard` enforces `@Roles('seller' | 'admin')` on protected routes

**Seed (`prisma/seed.ts`):**

- 1 seller: `seller@test.com` / `password123`
- 1 admin: `admin@test.com` / `password123`

**Web:**

- `/login` page with email + password form (see `docs/prototypes/login.html`)
- On success: `role === 'seller'` → redirect `/seller/documents`; `role === 'admin'` → redirect `/admin/documents`
- Zustand auth store: `{ userId, role, name, email }`, persisted to localStorage
- `lib/api.ts` Axios interceptor attaches `Authorization: <userId>` to every request
- `/_seller` route group: `beforeLoad` checks `role === 'seller'`, else redirect `/login`
- `/_admin` route group: `beforeLoad` checks `role === 'admin'`, else redirect `/login`

## Acceptance criteria

- [x] `POST /auth/login` returns 200 + `{ userId, role, name, email }` for valid credentials
- [x] `POST /auth/login` returns 401 for invalid credentials
- [x] Seller login redirects to `/seller/documents`
- [x] Admin login redirects to `/admin/documents`
- [x] Direct navigation to `/admin/*` by a seller redirects to `/login`
- [x] `userId` stored in Zustand, sent as `Authorization: <userId>` on subsequent requests
- [x] Seed runs via `pnpm prisma db seed` without errors
